INK

Ink was my group's final project for CS354R: Game Technology with Dr. Abraham. It is a survival horror game focused around a single advanced AI chasing the player around a 2D sidescrolling level (we based this AI heavily around Mr. X from Resident Evil 4, specifically in terms of it's lack of omnipotense regarding information about the player).
The main features that I worked on as a member of this team were:
  - All Art Assets (minus the background texture for the demo level)
  - All Animations, and the Animation State Machine
  - The Entire Player Package
  - Level Design & Lighting
  - Djikstra's Room Graph Algorithm
The monster moves via Djikstra's Shortest Path Algorithm to navigate the level. We effectively modeled the entire level as a giant graph of "Room Nodes" that the monster can traverse between. This allows for very interesting behavior as it allows for the monster to easily make complex logical decisions when chasing the player about how best to navigate to them in the shrotest number of "Room Traversals" possible (obviously weighted by room size).
Overall, this design was very effective for achieving the AI technology we wanted, but it made level design very cumbersome due to needing to set up the "metadata" for the graph every time you wanted to add more rooms. Given more time, a tool to automate this process for level designers would have extremely useful and a much needed addition to the project.
